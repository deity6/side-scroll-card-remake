# 更新日志

本文件记录面向版本发布的主要改动。更细的设计与提示词资料见 `docs/ai_prompts/`。

## v0.3.2 - 2026-06-21

### 新增

- 新增春日森林版战斗场景素材，包含背景、玩家底部栏、敌人状态框、手牌阴影、牌库图标、行动力图标、退出战斗按钮与结束回合按钮。
- 新增手绘简约萌宠风格 battle card 分层素材，包含卡牌底板、边框、费用图标、类型标签、描述框、选中高亮、禁用遮罩与卡背。
- 新增 `docs/ai_prompts/05_战斗场景与卡牌素材.md`，记录战斗场景、卡牌素材、手牌层级和 z-index 规划。
- 新增 `docs/ai_prompts/版本记录.md`，用于记录阶段版本内容和后续任务。

### 调整

- 将当前原型版本定义为 `v0.3.2`，同步更新主菜单版本号与 `README.md`。
- 重构 `scenes/battle_card.tscn` 为固定 `240x360` 的卡牌分层结构。
- 更新 `scripts/battle_card.gd`，支持按卡牌类型切换底板与边框。
- 更新 `scenes/battle_scene.tscn`，接入春日森林战斗 UI 素材。
- 调整 `scripts/battle_scene.gd` 的手牌渲染和飞入动画层级。
- 重新规划战斗场景 z-index，总层数控制在 `0..14`，避免继续使用 `1000/10000` 等过高层级。
- 将 `PlayerArea` 调整为更适合手动摆放子节点的布局结构，便于在 Godot 编辑器中直接调整控件位置。
- 整理 `docs/ai_prompts/AI_PROMPTS_INDEX.md`、`04_美术资产.md`、`ui设计.md` 和 `注意事项.md`，让当前素材落点和约束更清晰。

### 修复

- 修复卡牌堆叠时文字或局部图层穿插遮挡的问题：卡牌内部不再依赖高 z-index，整张卡作为独立渲染单元参与手牌堆叠。
- 修复抽牌/发牌飞入动画层级不足的问题：飞入中的卡牌临时处于前景层，落位后回到正常手牌层。
- 修复旧手牌位置配置中历史乱码注释导致的可维护性问题。

### 已知后续工作

- 敌人立绘仍为占位资源，后续需要确定 Spine、GIF 或序列帧方案。
- buff/debuff、玩家装备与技能图标目前仍以模板资源为主，后续需要接入运行时刷新。
- `Data/cards/*.json` 中仍有历史乱码数据，后续应集中清理并统一卡牌文本规范。

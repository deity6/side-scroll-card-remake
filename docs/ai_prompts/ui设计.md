# UI 设计风格指南

## 当前主题

- 主题：手绘简约萌宠 + 森林冒险 + 春机盎然。
- 基调：清爽、柔和、木质、纸面、叶片点缀。
- 使用场景：第一章 battle scene、battle card、玩家 HUD、战斗按钮。

## 布局原则

- 战斗界面优先保证可读性，装饰元素不能压住血条、行动力、手牌和按钮。
- 可手动摆放的复杂视觉区域使用 `Control`，避免 `Container` 抢占子节点位置。
- 需要动态排列的列表可以使用 `HBoxContainer` 或 `VBoxContainer`，例如装备列表、Debug 工具条。
- 贴图层和逻辑层分离：`TextureRect` 做视觉，`Button` 做交互。

## 卡牌 UI

- 标准尺寸：`240x360`。
- 卡牌内部靠节点顺序渲染，不使用高 z-index。
- 卡牌作为整体参与手牌堆叠，避免上一张卡的文字或边框穿透下一张卡。
- 文字使用固定区域，不使用自动撑开布局。
- `RichTextLabel` 应关闭 `fit_content`，固定区域并裁剪越界文本。

## 战斗场景 z-index

总层数控制在 `0..14`：

- `0`：背景与默认贴图
- `1`：敌人区域
- `2`：玩家底部背景
- `4..13`：手牌堆叠
- `14`：飞入、选中、拖拽、Debug、结算面板

不要使用大跨度 z-index。需要“从前景飞入”的对象使用当前区域内的最高前景层，而不是使用 `1000` 或 `10000`。

## 按钮与反馈

- 按钮可使用透明 `Button` 覆盖在贴图上。
- 按钮贴图应有 normal、pressed、disabled 的扩展空间。
- Debug 按钮保持功能性优先，不强制套正式美术。

## 后续 UI 方向

- 补充敌人 buff/debuff 图标运行时刷新。
- 补充玩家技能按钮和技能图模板。
- 补充卡牌稀有度、升级态、选中态和不可用态细节。

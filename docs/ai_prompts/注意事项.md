# 注意事项

## Godot 4.4.1 兼容性
- 本项目使用 Godot_v4.4.1-stable_mono_win64，不支持 4.5+ 特性。
- 禁止使用 ScriptBacktrace 类型（4.5+ 新增）。
- 禁止使用 OS.add_logger() 静态函数（4.5+ 新增）。
- .tscn 和 .gd 文件不能带 UTF-8 BOM，否则 Godot 解析报错。
- 写入文件时使用 Python 写入，不要用 PowerShell 的 Set-Content -Encoding UTF8。

## GDScript 缩进
- 统一使用 4 空格缩进，不使用 Tab。
- if/else/for/while 块内部必须有缩进，不能留空块。

## 场景文件（.tscn）
- 需要通过百分号节点名访问的节点，必须设置 unique_name_in_owner = true。
- 不要在场景的 connection 和脚本的 .connect() 中重复连接同一个信号。
- 推荐：信号全部由脚本管理，场景中不写 connection 行。

## Autoload 使用
- Autoload 节点在 _ready() 中通过 get_node 获取实例。
- 不要用 preload 把实例脚本当静态类调用。

## godot-mcp 插件
- 该插件声明需要 Godot 4.5+，与当前 4.4.1 不完全兼容。
- mcp_game_bridge.gd 已被简化为空脚本以避免编译错误。
- 如需完整 MCP 功能，请升级到 Godot 4.5+。

## Git 注意
- .godot/ 目录不应提交到版本控制。
- 每次修改 .tscn 后确认无 BOM 再提交。

## PowerShell 中文乱码问题
- PowerShell 5.x 的 heredoc (@' ... '@) 通过管道传给 Python 时，部分中文字符会丢失或变成问号。
- 原因：PowerShell 管道默认使用系统编码（GBK/CP936）传输字节，而 Python 期望 UTF-8，导致多字节中文字符被截断。
- 受影响场景：用 PowerShell 的 Set-Content 或 heredoc 写入含中文的 .gd/.tscn 文件。
- 不受影响：直接用 Python 的 pathlib.write_text(encoding=utf-8) 写入中文是正常的。
- 解决方案：需要写入中文内容时，优先用 Python 脚本（通过 py -c 或 .py 文件）写入，避免 PowerShell 管道传输中文。
- 已知安全方式：用 Unicode 转义序列（如 \u4e2d\u6587 代表“中文”）在 PowerShell heredoc 中传递给 Python 可以避免乱码。

## 换行符与缩进一致性
- 同一个文件中不可混用 CRLF 和 LF 换行符。
- 推荐统一使用 LF（\n）。
- 同一个文件中不可混用 Tab 和空格缩进，统一使用 4 个空格。
- 函数之间保留最多 1 个空行，不要连续多个空行。

## 代码与场景同步
- 修改代码后，如果影响了场景文件（.tscn）中的属性（如节点名、初始值、节点类型等），必须及时同步场景文件。
- 例如：代码中新增了一个 @onready 引用的节点，但场景中没有该节点或缺少 unique_name_in_owner，运行时会报错。
- 例如：代码中修改了持久化值（如版本号），场景中的初始值也应同步更新。
- 每次提交前检查：代码中的引用、变量名、初始值是否与 .tscn 场景文件一致。

# 注意事项

## 文件操作方式（强制）

- **所有文件的读取、修改、搜索、终端命令执行，必须且只能通过 `desktop-commander` MCP 工具完成。**
- 常用工具：`mcp__desktop_commander__read_file`（读文件）、`mcp__desktop_commander__edit_block`（精确编辑）、`mcp__desktop_commander__write_file`（写文件）、`mcp__desktop_commander__list_directory`（列目录）、`mcp__desktop_commander__start_search`（搜索）、`mcp__desktop_commander__start_process`（终端命令）。
- 禁止使用内置的 `exec_command`、`read_mcp_resource`（文件类）等其他方式读写项目文件。

## Git 操作方式（强制）

- **所有 git 命令必须通过 `desktop-commander` MCP 的 `start_process` 工具调用 Git Bash 执行。**
- Git Bash 路径：`C:\Program Files\Git\bin\bash.exe`
- **禁止使用 PowerShell 或 CMD 执行 git 命令**，原因：
  - PowerShell 会破坏中文路径编码，导致 git 无法识别文件。
  - Codex 内置沙盒对 `.git` 目录有写入权限限制，PowerShell 走沙盒会报 `Permission denied`。
  - Git Bash 不受上述限制，能正确处理中文路径和 `.git` 写入。
- 标准调用模板：
  ```
  mcp__desktop_commander__start_process(
    command: "cd \"D:/Got/GodotSharp/Program/卡牌复刻\" && git <命令>",
    shell: "C:\\Program Files\\Git\\bin\\bash.exe",
    timeout_ms: 30000
  )
  ```
- 常用 git 操作示例：
  - 提交推送：`cd "D:/Got/GodotSharp/Program/卡牌复刻" && git add <文件> && git commit -m "消息" && git push origin master`
  - 查看状态：`cd "D:/Got/GodotSharp/Program/卡牌复刻" && git status`
  - 查看日志：`cd "D:/Got/GodotSharp/Program/卡牌复刻" && git log --oneline -5`
  - 暂存还原：`cd "D:/Got/GodotSharp/Program/卡牌复刻" && git stash && git stash pop`
  - 回退文件：`cd "D:/Got/GodotSharp/Program/卡牌复刻" && git checkout HEAD -- <文件>`
  - 打标签：`cd "D:/Got/GodotSharp/Program/卡牌复刻" && git tag -a v0.x.x -m "版本说明" && git push origin master --tags`

## Godot 版本

- 当前项目使用 Godot 4.4.1。
- 禁止使用 Godot 4.5+ 新增 API。
- `.gd/.tscn` 使用 UTF-8 无 BOM 和 LF 换行。
- GDScript 使用 Tab 缩进。

## 文档注释

- README、设计文档和提示词文档中，如果提到具体英文定义的文件、脚本、场景、节点、方法名、属性名或资源目录，应附带简短中文注释。
- 中文注释应说明"它是什么/负责什么"，不要只翻译单词。
- 目录树、关键节点列表、方法列表和配置表必须优先补中文说明，方便后续维护时直接理解用途。

## 场景与脚本同步

- 修改节点名、节点类型、`@onready` 引用或 `unique_name_in_owner` 时，必须同步检查 `.tscn` 与 `.gd`。
- 场景里的 `connection` 和脚本里的 `.connect()` 不要重复连接同一信号。
- `@onready` 变量在 `_ready()` 前可能为空，外部提前调用的 setter 需要空值保护。

## Autoload

- Autoload 实例通过 `get_node("/root/Name")` 获取。
- 不要把 Autoload 脚本 `preload` 后当静态类调用。

## Debug 逻辑

- Debug 按钮必须复用正式逻辑。
- 例如 debug 加牌调用 `_draw_cards_to_hand()`，而不是新写独立抽牌路径。
- Debug 的作用是模拟玩家操作来验证正式流程。

## 重要改动备份

- 删除或大规模修改关键文件前，先通过 Git Bash 执行 `git stash` 或 `git commit` 做本地备份。
- Godot 编辑器运行时可同时修改文件，Godot 会弹出"重新加载"提示，点击确认即可同步。

## 音频系统

- 音效通过 SoundManager.play_sfx("key") 或 SoundManager.play_sfx_varied("key") 调用。
- 音效映射配置在 Data/sfx_mapping.json，新增音效只需添加映射和素材文件。
- BGM 通过 SoundManager.play_bgm("key") / stop_bgm() 调用（预留接口，素材待后续添加）。
- AudioBus 分三条：Master -> SFX / BGM，配置在 default_bus_layout.tres。
- 音效素材统一使用 MP3 格式，存放在 assets/audio/sfx/ 按场景分类目录下。

## 安全与发布

- 禁止把密码、密钥、API Key、`.env`、证书、私钥、访问令牌写入代码、文档或提交到仓库。
- `.gitignore` 必须忽略 `.env*`、`*.pem`、`*.key`、`*.pfx`、`*.p12`、`*.crt` 和 `export_presets.cfg`。
- `export_presets.cfg` 可能包含签名、证书或平台账号相关配置，除非确认无敏感信息，否则不要提交。
- `addons/godot_mcp` 仅作为开发期辅助插件使用；正式导出或发布前应确认不会开启远程调试/命令桥接能力。
- MCP 插件绑定地址默认应保持本机地址；不要在不可信网络下绑定 `0.0.0.0`、公网地址或未确认的局域网地址。
- 不要把用户输入、项目配置、环境变量拼接进 `OS.execute()`、PowerShell、cmd 或 shell 命令。
- 运行时存档只写入 `user://`，不要写入 `res://` 或项目根目录。

## 单一权威来源

- 每个数据只能有一个权威默认值来源。
- 玩家属性：`adventure_player_state.gd`
- 战斗传递：`global_battle_request.gd`
- 战斗运行：`battle_state.gd`
- 卡牌定义：`card_pool.gd` + `Data/cards/*.json`

## UI 与 z-index

- 战斗场景总 z-index 控制在 `0..14`。
- 不要使用 `1000`、`10000` 等大层级。
- 卡牌内部不使用高 z-index，卡牌应作为整体参与手牌堆叠。
- 复杂视觉区需要手动拖动时使用 `Control`，不要把子节点放在会自动排版的 `Container` 下。

## 图片上下文

- 用户发图后，只需解析一次关键内容。
- 后续工作应基于已提取的信息继续，不要反复重新描述图片。

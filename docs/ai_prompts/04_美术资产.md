# 美术资产规范

## 风格方向

- 当前主风格：手绘简约萌宠 + 森林冒险 + 春机盎然。
- 视觉关键词：圆润、木质边框、嫩绿叶片、浅暖色纸面、低压迫感、适合竖屏移动端阅读。
- 第一章场景避免雪天、暗黑和复杂堆叠元素，优先体现春日森林的清爽空间。

## 资源目录

- `assets/ui/battle_scene_pet_spring/`：战斗场景 UI 正式素材。
- `assets/cards/battle_card_pet/`：battle card 正式素材。
- `assets/card_template/`：旧卡牌模板和缩放模板。
- `assets/ai制作/`：AI 制作过程稿、备用图、透明处理结果，不作为首选运行时引用。

## 命名规范

- 使用英文 snake_case。
- 素材名应体现用途、类型和状态，例如 `btn_end_turn_normal.png`。
- 运行时稳定引用的资源不要频繁改名。
- 过程文件可以保存在 `assets/ai制作/`，正式引用应复制到 `assets/ui/` 或 `assets/cards/`。

## 导出规范

- UI 和卡牌分层素材优先使用透明 PNG。
- 如果需要抠图，源图可使用高反差背景色，但最终进入正式目录前应处理为可直接引用的 PNG。
- 背景图不需要抠透明，保持完整画面。
- 卡牌标准运行时尺寸为 `240x360`，源图可更大，但场景中需按固定尺寸约束。

## 工作流

1. 先生成或整理过程素材到 `assets/ai制作/`。
2. 挑选正式版本复制到 `assets/ui/` 或 `assets/cards/`。
3. 场景文件只引用正式路径。
4. 更新 `docs/ai_prompts/05_战斗场景与卡牌素材.md` 和索引。

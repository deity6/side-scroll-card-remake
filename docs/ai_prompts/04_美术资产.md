# 美术资产规范

## 风格方向
- 参考《月圆之夜》童话手绘风格。
- 适配竖版可读性：大对比、清晰边线、少噪点。
- 暗色调为主，关键信息用亮色强调。

## 资产清单

### 角色
- 女骑士：`idle`、`attack`、`defend`、`hurt`、`death`。
- 敌人（每种）：`idle`、`attack`、`hurt`。

### 卡牌
- 每张卡牌：卡面图 + 边框。
- 尺寸：256x384px（竖版比例）。
- 格式：PNG，透明背景。

### UI
- 可交互按钮：按下时缩放到 0.95，松手弹回 1.0，使用 `TRANS_ELASTIC` 果冻回弹动画。
- 可交互信息（如金币图标被长按）：添加简单“被选中”状态效果，例如轻微加亮。
- 血条：填充条 + 数字。
- 能量珠：圆形，亮/暗两种状态。
- 按钮：正常、悬停、按下、禁用四态。
- 意图图标：攻击、防御、技能、未知。

### 地图
- 节点图标：战斗、精英、商店、事件、休息、Boss。
- 连线：已走过、可选择、未解锁。

## 命名规范
- 英文蛇形命名：`knight_idle.png`、`card_slash.png`。
- 目录结构：
  - `assets/characters/`
  - `assets/cards/`
  - `assets/ui/`
  - `assets/map/`
  - `assets/effects/`

## 导出规范
- 透明背景 PNG。
- 无损压缩。
- 锚点统一为底部中心（角色立绘）。
- 卡牌锚点为顶部中心。

## 工作流
1. 先用纯色矩形占位，验证布局。
2. 用简易图标（圆形+文字）验证交互。
3. 最后替换正式美术资源，不改游戏逻辑。

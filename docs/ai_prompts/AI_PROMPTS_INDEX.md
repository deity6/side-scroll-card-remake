## 文档目录

### `docs/ai_prompts/注意事项.md`

- 作用：工程约束与兼容性规则。
- 内容：文件操作方式（desktop-commander MCP 强制）、**Git 操作方式（Git Bash 强制）**、Godot 4.4.1 兼容、GDScript 缩进、英文定义中文注释、`.tscn/.gd` 同步、Autoload、Debug 逻辑复用、音频系统、安全与发布等。
- 适用场景：任何代码或场景改动前都应先确认。**执行 git 操作前必须确认使用 Git Bash。**

### `docs/ai_prompts/00_总控提示词.md`

- 作用：项目总纲与阶段路线。
- 内容：项目定位、阶段目标、目录约定、工作顺序和验收口径。
- 适用场景：开始新阶段、拆任务、写 PR/提交说明。

### `docs/ai_prompts/01_战斗手感增强.md`

- 作用：战斗体验设计规范。
- 内容：输入层、时间层、反馈层、信息层、竖屏布局与动效节奏。
- 适用场景：出牌交互、受击反馈、意图提示、HUD 布局、动效节奏。

### `docs/ai_prompts/02_肉鸽流派感.md`

- 作用：卡组构建与关卡节奏设计。
- 内容：流派标签、卡池生成、章节节奏、敌人克制与验收标准。
- 适用场景：卡池规划、奖励生成、Boss 机制、难度曲线。

### `docs/ai_prompts/03_局外养成.md`

- 作用：局外成长与存档规则。
- 内容：金币、经验、天赋、职业解锁、存档结构、数值边界。
- 适用场景：永久进度、解锁系统、存档读写、局外数值平衡。

### `docs/ai_prompts/04_美术资产.md`

- 作用：美术资产规范与工作流。
- 内容：风格方向、资源目录、命名规范、导出规范和当前素材落点。
- 适用场景：资源命名、目录建设、占位图替换、美术交付。

### `docs/ai_prompts/05_战斗场景与卡牌素材.md`

- 作用：v0.3.2 战斗场景和 battle card 素材接入规范。
- 内容：正式资源路径、`battle_scene` 节点原则、`battle_card` 分层关键节点中文注释、15 层以内 z-index 规划和手牌飞入规则。
- 适用场景：继续调整战斗场景、卡牌 UI、手牌动画和渲染层级。

### `docs/ai_prompts/ui设计.md`

- 作用：UI 风格与组件动效指南。
- 内容：春日森林主题、按钮/HUD/卡牌文字层级、容器与手动布局选择、交互反馈。
- 适用场景：冒险界面、卡牌、HUD、按钮反馈、节点状态视觉。

### `docs/ai_prompts/版本记录.md`

- 作用：记录阶段版本完成内容。
- 内容：v0.3.4 音效系统构建、v0.3.2 素材接入、卡牌分层、z-index 优化和后续工作。
- 适用场景：版本回顾、提交说明、后续任务衔接。
